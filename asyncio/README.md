# YCrawler

Асинхронный краулер для сайта https://news.ycombinator.com.

Программа периодически проверяет главную страницу Hacker News, скачивает топ-30 новостей и сохраняет:

* страницу новости
* страницу комментариев
* страницы по ссылкам, найденным в комментариях

Каждая новость сохраняется в отдельную директорию.

---

# Требования

* Python 3.7+
* aiohttp
* aiofiles
* beautifulsoup4

Установка зависимостей:

```bash
pip install aiohttp aiofiles beautifulsoup4
```

---

# Запуск

```bash
python crawler.py
```

По умолчанию:

* директория сохранения: `data`
* интервал проверки: `30` секунд

---

# Параметры

```bash
python crawler.py --out data --interval 30
```

### Аргументы

| аргумент     | описание                                   |
| ------------ | ------------------------------------------ |
| `--out`      | директория для сохранения страниц          |
| `--interval` | интервал проверки новых новостей (секунды) |

---

# Пример

```bash
python crawler.py --out data --interval 20
```

Краулер будет каждые 20 секунд проверять появление новых новостей.

# Структура сохранённых данных

После скачивания структура будет выглядеть так:

```
data/
  38837472_show_hn_my_ai_tool/
      story.html
      hn_comments.html
      comment_link_001.html
      comment_link_002.html
      comment_link_003.html
```

Описание файлов:

| файл                  | описание                            |
| --------------------- | ----------------------------------- |
| `story.html`          | страница статьи                     |
| `hn_comments.html`    | страница комментариев Hacker News   |
| `comment_link_*.html` | страницы по ссылкам из комментариев |

---

# Принцип работы

Краулер работает следующим образом:

1. скачивает главную страницу Hacker News
2. парсит топ-30 новостей
3. определяет новые новости, которых ещё не скачивали
4. для каждой новости:

   * скачивает страницу комментариев
   * извлекает ссылки из комментариев
   * скачивает страницу статьи
   * скачивает страницы по ссылкам из комментариев
5. сохраняет все файлы в отдельную папку
6. ждёт `N` секунд и повторяет процесс

# Используемые библиотеки

* **aiohttp** — асинхронные HTTP запросы
* **aiofiles** — асинхронная запись файлов
* **BeautifulSoup** — парсинг HTML

---

# Особенности

* асинхронная загрузка страниц
* параллельное скачивание ссылок из комментариев
* фильтрация некорректных ссылок
* защита от повторной загрузки новостей
* логирование через `logging`

---

# Ограничения

В соответствии с заданием:

* краулер **не ищет новые ссылки внутри скачанных страниц**
* скачиваются **только страницы из комментариев**
